Helpers (verbatim from research Code Examples):

```python
def log_scale(value: float, ref: float) -> float:
    """0 at value=0, 1.0 at value>=ref, log-shaped between."""
    return min(1.0, math.log10(1 + value) / math.log10(1 + ref))

def percentile(pool, value: float) -> float:
    """pool sorted ascending; self-inclusive mid-rank; deterministic under ties."""
    less = sum(1 for x in pool if x < value)
    equal = sum(1 for x in pool if x == value)
    return (less + 0.5 * equal) / len(pool)

def fmt_price(p: float) -> str:
    """Fixed-decimal, never scientific: 1.5e-05 -> '0.000015', 0.0 -> '0'."""
    return f"{p:.8f}".rstrip("0").rstrip(".")

def _median(pool) -> float:
    """pool sorted ascending, non-empty; even length averages the two middles."""
    n, mid = len(pool), len(pool) // 2
    return pool[mid] if n % 2 else (pool[mid - 1] + pool[mid]) / 2
```

Every component returns the SAME frozen dict shape: `{"score": int|None, "weight": WEIGHTS[name], "observed": ..., "benchmark": ..., "flagged": bool, "reason": str}`. `score: None` is the explicit insufficient-data state (never fabricate a number). `flagged` is False everywhere except C3's thin-perfect case. Reason strings are TEMPLATES + numbers + category bucket names ONLY — NEVER interpolate agent name, tagline, or any listing text (a real tagline, id 2361, contains a banned substring; this is a defamation-surface rule, not a style rule). All checks on rating/positive_pct/price_usdt use `is None` / `is not None` — never truthiness (0.0 is real data).

**C1 `c_sales_volume_velocity(sold: int, distinct_snapshots: int) -> dict`** (weight 0.30):
- `score = round(100 * log_scale(sold, SOLD_REF))` — always scored (sold=0 -> 0).
- velocity note: if `distinct_snapshots < 2` append `"; sales velocity unavailable — insufficient history (single snapshot)"` else append `"; sales velocity not computed in this score version"` (v2 sold-delta design is a docstring seam only — do NOT implement).
- reason: sold==0 -> `"no completed sales observed in the snapshot" + note`; else `f"{sold} unit(s) sold (volume scale tops out at 500+)" + note`.
- observed=sold, benchmark=SOLD_REF.

**C2 `c_review_signal_ratio(rating, positive_pct, sold, stats) -> dict`** (weight 0.20). Branch order is frozen (the None-pct check precedes the thin check):
1. rating is not None and positive_pct is None -> score 70, reason `f"rating displayed; positive-review share not shown on the listing ({sold} sales)"` (real case: id 2169).
2. rating is not None, pct present, sold < THIN_SALES -> `score = min(round(40 + 60 * positive_pct / 100), 65)`, reason `f"rating displayed with {positive_pct:g}% positive on only {sold} sale(s) — limited volume behind the review signal"`.
3. rating is not None, pct present, sold >= THIN_SALES -> `score = round(40 + 60 * positive_pct / 100)`, reason `f"rating displayed with {positive_pct:g}% positive across {sold} sales"`.
4. rating is None and sold >= RATING_EXPECTED_SALES -> score 35, benchmark=stats.rating_display_pct, reason `f"no displayed rating despite {sold} units sold — {stats.rating_display_pct}% of agents with 20+ sales display one ({stats.rated_hi} of {stats.total_hi})"`.
5. rating is None, sold < RATING_EXPECTED_SALES -> score None, reason `f"insufficient data — no displayed rating and only {sold} sale(s)"`.
- observed=positive_pct (may be None); benchmark=stats.rating_display_pct in branch 4 else None.

**C3 `c_rating_credibility(rating, sold) -> dict`** (weight 0.25):
- rating is None -> score None, reason `"insufficient data — no rating displayed to evaluate"`.
- else `support = log_scale(sold, SUPPORT_REF)`; `score = round(100 * (rating / 5.0) * (CRED_FLOOR + (1 - CRED_FLOOR) * support))`. The CRED_FLOOR exists because the floorless v1 ranked a rated agent below an identical unrated one — do not remove it.
- `flagged = (rating == 5.0 and sold < THIN_SALES)` (brief-verbatim case). Flagged reason: `f"perfect 5.0 rating backed by only {sold} sale(s) — pattern consistent with limited review history; low confidence, flagged for thin data (not an assessment of conduct)"`.
- not flagged: sold >= SUPPORT_REF -> `f"{rating:g}/5 rating supported by {sold} sales"`; else `f"{rating:g}/5 rating with moderate volume behind it ({sold} sale(s))"`.
- observed=rating, benchmark=5.0.

**C4 `c_price_vs_category(price_usdt, category, stats) -> dict`** (weight 0.15):
- price_usdt is None -> score None, reason `"insufficient data — no listed price to compare against category"` (28 real rows).
- pool = `stats.category_pools.get(category, ())`; label = category; if `len(pool) < MIN_CATEGORY_PRICED`: pool = stats.market_pool, label = `"marketplace"` (real trigger: "Other Services", 3 priced).

- observed=price_usdt, benchmark=`round(med, 6)` (None when insufficient).

```python
GRADE_BANDS = (("A", 85), ("B", 70), ("C", 55), ("D", 40), ("F", 0))  # first match, score >= cut
HIGH_CONF_SOLD = 50
LOW_CONF_SOLD = 5
DISCLAIMER = (

    "data as of the stated snapshot; it is not a statement of fact about any "
    "vendor or agent."
)
GRADE_DESCRIPTIONS = {
    "A": "high sales volume with a strong, well-supported displayed rating",
    "B": "solid sales volume with a strong rating, or exceptional volume without a displayed rating",
    "C": "modest sales volume with a displayed rating",
    "D": "thin evidence — limited sales or limited review signal behind the listing",
    "F": "minimal evidence or negative review signals in the observed data",
    "NR": "not rated — no transaction or review evidence observed yet",
}
```

```python
def grade_for(score: int) -> str:
    return next(g for g, lo in GRADE_BANDS if score >= lo)

def serialize_components(components: dict) -> str:
    return json.dumps(components, sort_keys=True, separators=(",", ":"), ensure_ascii=False)

def score_agent(row: Mapping, stats: Stats, distinct_snapshots: int) -> dict:
    """Pure: no I/O, no wall clock. row needs keys category, sold, rating,
    positive_pct, price_usdt, first_seen."""
    sold, rating = row["sold"], row["rating"]
    components = {
        "sales_volume_velocity": c_sales_volume_velocity(sold, distinct_snapshots),
        "review_signal_ratio": c_review_signal_ratio(rating, row["positive_pct"], sold, stats),
        "rating_credibility": c_rating_credibility(rating, sold),
        "price_vs_category": c_price_vs_category(row["price_usdt"], row["category"], stats),
        "listing_age_consistency": c_listing_age_consistency(row["first_seen"], distinct_snapshots),
    }
    # NR rule (verbatim, research-locked): zero transaction evidence AND zero
    # review evidence -> honest not-rated state; components still rendered.
    if sold == 0 and rating is None:
        return {"score": None, "grade": "NR", "confidence": "low", "components": components}
    scored = {k: c for k, c in components.items() if c["score"] is not None}
    total_w = sum(WEIGHTS[k] for k in scored)  # renormalize over available evidence
    score = round(sum(WEIGHTS[k] * c["score"] for k, c in scored.items()) / total_w)
    grade = grade_for(score)
    flagged = components["rating_credibility"]["flagged"]
    if flagged or sold < LOW_CONF_SOLD or len(scored) <= 2:
        confidence = "low"
    elif len(scored) >= 4 and sold >= HIGH_CONF_SOLD and rating is not None:
        confidence = "high"
    else:
        confidence = "medium"
    return {"score": score, "grade": grade, "confidence": confidence, "components": components}
```

```python
"""Read-only DB access for the MCP tools. Connection-per-call; never writes."""
from __future__ import annotations

import sqlite3
from pathlib import Path

from indexer.parse import name_key

DEFAULT_DB = Path("data/trustlens.db")

CARD_SELECT = (
    "SELECT a.id, a.name, a.category, a.tagline, a.price_usdt, a.sold, a.rating,"
    " a.positive_pct, s.score, s.grade, s.confidence, s.score_version,"
    " s.generated_at, s.data_as_of, s.components"
    " FROM agents a JOIN scores s ON s.agent_id = a.id"
)


def connect_ro(db_path: str | Path = DEFAULT_DB) -> sqlite3.Connection:
    # research-verbatim: as_uri() percent-encodes the spaces in this repo's path
    uri = Path(db_path).resolve().as_uri() + "?mode=ro"
    conn = sqlite3.connect(uri, uri=True)
    conn.row_factory = sqlite3.Row
    return conn
```

Functions (each takes `conn` as first arg so tools own the connection lifecycle):

1. `lookup_agent(conn, query: str) -> tuple[sqlite3.Row | None, list[sqlite3.Row]]` — the locked ladder, returning `(row, ambiguous)`:
   - Step 1: `CARD_SELECT + " WHERE a.id = ?"` with `(query,)` — hit -> `(row, [])`
   - Step 2: `... WHERE a.name = ? ORDER BY a.id` — 1 row -> `(row, [])`; 2+ -> `(rows[0], rows)` (defensive; names are unique in the real census)
   - Step 3: `... WHERE a.name_key = ? ORDER BY a.id` with `(name_key(query),)` — 1 row -> `(row, [])`; 2+ rows -> `(rows[0], rows)` (LOWEST id wins deterministically, ALL matches returned for disclosure); 0 rows -> `(None, [])`
2. `closest_candidates(conn, query: str, limit: int = 3) -> list[sqlite3.Row]` — deterministic not-found suggestions: prefix match on name_key with LIKE wildcards ESCAPED in the user input (`q = name_key(query).replace("\\", "\\\\").replace("%", r"\%").replace("_", r"\_")` then `WHERE name_key LIKE ? ESCAPE '\'` with `q + "%"`, `ORDER BY id LIMIT ?`). Returns `(id, name)` rows; may be empty
3. `category_slice(conn, category: str, limit: int) -> list[sqlite3.Row]` — scored agents only: `SELECT a.id, a.name, s.score, s.grade, s.confidence FROM agents a JOIN scores s ON s.agent_id = a.id WHERE a.category = ? AND s.score IS NOT NULL ORDER BY s.score DESC, a.id LIMIT ?`
4. `category_total(conn, category: str) -> int` — `SELECT COUNT(*) FROM agents WHERE category = ?` (scored + NR)
5. `stats(conn) -> dict` — computed via SQL over agents+scores: `agents_total` (COUNT agents), `scored` (COUNT scores WHERE score IS NOT NULL), `not_rated` (WHERE score IS NULL), `grade_distribution` (GROUP BY grade -> dict with keys in sorted order), `category_counts` (GROUP BY category -> dict with keys in sorted order), `median_price_usdt` (fetch all non-NULL price_usdt ordered, `statistics.median` in Python; None when empty)
6. `envelope_values(conn) -> sqlite3.Row | None` — `SELECT score_version, MAX(generated_at) AS generated_at, MAX(data_as_of) AS data_as_of FROM scores GROUP BY score_version LIMIT 1` (all rows share one version/timestamp — verified); None when scores table is empty

Module docstring notes the locked rule: this module NEVER writes and NEVER recomputes scores; timestamps are SERVED from storage (no wall clock — the MCPS-02 determinism guarantee rides on this).

class ScoreCard(TypedDict):
    agent_id: str
    name: str
    category: str
    score: int | None            # None for NR — anyOf[integer,null] in schema
    grade: str
    confidence: str
    score_version: str
    generated_at: str
    data_as_of: str
    methodology_url: str
    disclaimer: str

    agents: list[ScoreCard]

    methodology_url: str
    score_version: str

class CategoryLeaderboard(TypedDict):

    generated_at: str
    methodology_url: str

**Envelope helpers:**
- `def _methodology_url() -> str: return os.environ.get("TRUSTLENS_BASE_URL", "http://localhost:8000") + "/#methodology"` (locked: env `TRUSTLENS_BASE_URL`, default `http://localhost:8000`; read at CALL time so tests can monkeypatch)
- `def _err(payload: dict) -> ToolError: return ToolError(json.dumps(payload, ensure_ascii=False, sort_keys=True, separators=(",", ":")))` — the ONLY error channel (research-verified: ToolError -> clean `isError:true`; plain exceptions LEAK internals verbatim to clients)

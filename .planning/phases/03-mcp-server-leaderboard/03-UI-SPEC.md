# Phase 3 — UI Design Contract

> Visual and interaction contract for the TrustLens leaderboard page (`/`) and the TrustLens Verified badge SVG (`/badge/{agent_id}.svg`). Generated by gsd-ui-researcher (autonomous mode — user away; unlocked choices use professional defaults marked "default — revisit if desired"), verified by gsd-ui-checker.

**Scope:** One static HTML page generated at build time by `web/build.py` from SQLite (WEB-01..03), plus one server-rendered SVG badge per agent. The page will be screen-recorded on a desktop display for hackathon judging — target aesthetic is analytics-dashboard / credit-report clarity, not a marketing site.

**Hard constraints (locked by 03-CONTEXT.md — non-negotiable):**
- Single static HTML page, inline CSS/JS, NO framework, NO external requests (no webfonts, no CDNs, no analytics, no images beyond inline SVG)
- No dark mode
- Neutral analytic grade colors — no alarmist red "danger" styling anywhere
- All copy factual, methodology-linked; banned vocabulary ("fraud", "scam", "fake", "manipulat") must never appear in page copy or template strings — extend the Phase 2 banned-word test to cover `web/build.py`
- Sortable table + category filter; "About the methodology" section at `id="methodology"`; badge embed snippet section
- Loads in <2s

The CJK members (PingFang SC / Microsoft YaHei / Noto Sans CJK SC) are mandatory — agent names include CJK (e.g. "这个能吃吗？") and must render in a matching sans, not a serif fallback.

All numeric table cells (`TrustScore`, `Sold`, `Rating`, `Price`) use `font-variant-numeric: tabular-nums` so digits align vertically.

**Radii:** 8px for cards/code block/buttons/select; 999px for grade chips; 3px for badge SVG corners. **No box-shadows anywhere** — 1px borders only (crisper under video compression; flat analytic look). *(default — revisit if desired)*

### Grade Badge System (chips — the core visual encoding)

Every band gets the same construction: pill chip, tinted background + dark text + 1px tinted border, letter(s) centered, 14px / 600, padding 4px 12px, min-width 36px, `border-radius: 999px`, `text-align: center`. The letter is the primary encoding; color is secondary reinforcement (color is never the sole signal — accessibility).

Hues are deliberately desaturated (Tailwind 100-level backgrounds, 800-level text). Nothing saturated, no warning icons, no "danger" styling. F uses a desaturated rose — consistent with credit-report convention that ordinal shading of published grade bands is factual presentation; there is no #DC2626-class alarm red anywhere on the page. Note: D is the largest scored band (54 of 121) and NR is over half the table (151 of 272), so the D apricot and NR gray were chosen to sit quietly in bulk.

| Grade | Background | Text | Border | Contrast |
|-------|-----------|------|--------|----------|
| A | #DCFCE7 (green-100) | #166534 (green-800) | #BBF7D0 | ≥7:1 |
| B | #CCFBF1 (teal-100) | #115E59 (teal-800) | #99F6E4 | ≥7:1 |
| C | #FEF9C3 (yellow-100) | #854D0E (yellow-800) | #FEF08A | ≥6.5:1 |
| D | #FFEDD5 (orange-100) | #9A3412 (orange-800) | #FED7AA | ≥7:1 |

**Confidence column:** plain lowercase text ("high" / "medium" / "low"), 14px / 400, #64748B. No chips, no colors, no icons — confidence is metadata, not a verdict.

## Page Architecture

Section order, top to bottom (single column):

1. **Header** — wordmark h1, subtitle, credibility line, inline nav anchors ("Methodology" · "Verified badge"). Not sticky.
2. **Controls row** — category `<select>` with visible label, live count readout, NR legend line.
3. **Leaderboard table** — white card (1px #E2E8F0 border, radius 8px), sticky `<thead>`.
4. **Badge section** — `id="badge"`: heading, one-line meaning statement, live example badge (inline SVG), copyable HTML + Markdown snippets, copy button.
5. **Methodology section** — `id="methodology"`: heading "About the methodology", prose + two small tables (components/weights, grade bands).
6. **Footer** — MCP funnel line, disclaimer verbatim, version/date stamp.

Vertical rhythm: 48px top padding on header; 64px between sections 3→4→5→6; 32px between controls and table.

Every table row carries `id="agent-{agent_id}"` — the badge embed links to `{TRUSTLENS_BASE_URL}/#agent-{id}` and `tr:target` gets the #EFF6FF highlight (pure CSS, zero JS).

## Table Contract

**Columns (exact order, headers, alignment):**

| # | Header | Alignment | Content / format |
|---|--------|-----------|------------------|
| 1 | `#` | right | Global TrustScore rank (precomputed at build). NR rows: "—" (unranked, always after scored rows in default order) |
| 2 | `Agent` | left | Agent name; wraps naturally, `max-width: 280px`; `title` attribute carries the tagline (native tooltip, no JS) *(tagline tooltip is default — revisit if desired)* |
| 3 | `Category` | left | One of the 9 buckets |
| 4 | `TrustScore` | right | Integer 0–100, tabular-nums; NR rows: "—" |
| 5 | `Grade` | center | Grade chip (system above) |
| 6 | `Confidence` | left | "high" / "medium" / "low" plain text |
| 7 | `Sold` | right | Thousands-separated integer ("1,550") |
| 8 | `Rating` | right | One decimal ("4.8"); null: "—" |
| 9 | `Price (USDT)` | right | See formatting rules; null: "—" |

No external links to okx.ai listing pages: the schema stores no listing URL and fabricating a URL pattern risks broken links on the judging recording — consciously omitted *(default — revisit if desired)*.

**Data formatting rules (applied in Python at build time — deterministic, never in JS):**
- Score: `str(int)`; NR → "—"
- Sold: `f"{sold:,}"`
- Rating: `f"{rating:.1f}"`; missing → "—"
- Price: `≥1` → `f"{p:.2f}"`; `<1` → `f"{p:.6f}".rstrip("0").rstrip(".")` (renders the census edge case 0.000015 exactly; never scientific notation); missing → "—"
- All null placeholders are em-dash "—" (U+2014)
- Every sortable cell carries `data-v` with the raw sort value (number, or grade/confidence ordinal, or codepoint string); missing values → `data-v=""`

**Sort semantics (vanilla JS, click on header):**
- Default order (initial page state and the `#` header's action): rank ascending = TrustScore desc, NR rows after scored, ties broken by agent id asc (matches the deterministic-output contract in FEATURES.md)
- Clicking any other header sorts by that column; second click reverses. Active header shows ↑ or ↓ glyph + accent color + `aria-sort="ascending|descending"`. Inactive headers show no arrow; hover bg #F1F5F9 and `cursor: pointer` signal affordance.
- Null/NR handling: empty `data-v` ALWAYS sorts last regardless of direction (an NR agent never outranks a scored one on TrustScore; a missing price never wins a price sort)
- Grade ordinal: A=5 B=4 C=3 D=2 F=1 NR=null-last. Confidence ordinal: high=3 medium=2 low=1
- Text columns (Agent, Category): raw codepoint comparison on the `data-v` string (deterministic across machines/locales; CJK sorts by codepoint) — never `localeCompare`
- All ties broken by agent id asc

**Category filter:**
- Native `<select>`, visible `<label>` "Category", options: "All categories" (default) + the 9 buckets alphabetically: Creative & Media, Developer Tools & Infra, Lifestyle & Health, Market Data & Analytics, Other Services, Security & Trust, Social & News, Sports & Prediction, Trading & DeFi
- Filtering toggles `display: none` per row (272 rows — no virtualization needed); count readout updates: "Showing {n} of 272 agents"
- Rank values do NOT renumber on filter or sort — rank is a static attribute of the agent (global TrustScore rank), which is the honest presentation

**Sticky header:** `thead th { position: sticky; top: 0; background: #FFFFFF; z-index: 1; border-bottom: 1px solid #E2E8F0; }` — background is mandatory or rows show through. Sticky degrades gracefully inside the small-screen `overflow-x: auto` wrapper; desktop (the demo) is the target.

**Row treatment:** `tr` bottom border 1px #F1F5F9; hover bg #F8FAFC; cell padding 8px 12px. No zebra striping *(default — revisit if desired)*.

## Badge SVG Contract (`/badge/{agent_id}.svg`)

Shields.io flat-style, two segments, fully self-contained (no external font fetch — SVG-safe generic stack):

- Geometry: `viewBox="0 0 110 20"`, height 20px; left segment 0–64px, right segment 64–110px; outer corners rounded `rx="3"` (clip both rects to the rounded outline)
- Left segment: bg #334155 (slate-700), text "TrustLens", white #FFFFFF, centered at x=32
- Right segment: bg = grade badge color (below), text "{grade} {score}" (e.g. "B 82"); NR agents: text "NR" only; white, centered at x=87
- Text: `font-family="Verdana,'DejaVu Sans',Geneva,sans-serif" font-size="11"`, `text-anchor="middle"`, baseline y=14
- Accessibility: `<title>TrustLens grade {grade}</title>` as first child; `role="img"` + `aria-label` on the root
- Badge right-segment colors (dark tones, all ≥4.5:1 with white text): A #166534 · B #115E59 · C #854D0E · D #9A3412 · F #9F1239 · NR #475569

*(fixed-width two-segment geometry is default — revisit if desired; self-hosted + grade-color + link-back are contract per FEATURES.md)*

**Embed snippet block (on the page, `id="badge"`):**
- Live example badge rendered inline (the real SVG markup for one real A-grade agent, inlined at build time — no self-request)
- Two copyable variants in one dark code block (bg #0F172A, text #E2E8F0, 16px padding, radius 8px, 14px mono, `overflow-x: auto`):

**Methodology section content (prose blocks + two tables, in order):**

1. Intro: "TrustScore is a 0–100 score computed by a deterministic, versioned formula over public OKX.AI marketplace data. The same data snapshot always produces the same score. There is no manual adjustment and no generated prose — and agents cannot pay to alter scores."
2. Components table (name / weight / what it measures):
   - Sales volume & velocity — 30% — units sold and growth across observed snapshots
   - Rating credibility — 25% — whether the displayed rating is supported by enough transactions
   - Review signal ratio — 20% — review and positive-percentage signal relative to units sold and category norms
   - Price vs category — 15% — price position within the agent's category (percentile)
   - Listing age & consistency — 10% — how long and how consistently the listing has been observed
   - Note under table: "When a component has no evidence for an agent, the remaining weights are renormalized rather than guessed."
3. Grade bands table (score range / grade / meaning — meanings verbatim from `scoring.GRADE_DESCRIPTIONS`):
   - A — 85–100 — "high sales volume with a strong, well-supported displayed rating"
   - B — 70–84 — "solid sales volume with a strong rating, or exceptional volume without a displayed rating"
   - C — 55–69 — "modest sales volume with a displayed rating"
   - D — 40–54 — "thin evidence — limited sales or limited review signal behind the listing"
   - F — 0–39 — "minimal evidence or negative review signals in the observed data"
   - NR — no score — "not rated — no transaction or review evidence observed yet"
   - Render each grade letter in this table as its chip (visual legend doubling as the band table)
4. NR rule: "An agent with zero recorded sales and no displayed rating is Not Rated (NR) rather than scored. NR is an honest insufficient-evidence state, not a low score. NR agents are listed after scored agents and are unranked."
5. Confidence: "Confidence (high / medium / low) reflects how much evidence backs a score. Thin evidence — for example a perfect rating with fewer than 5 recorded sales — lowers confidence and is flagged in the score card. A flag describes a statistical pattern; it is not an accusation."
6. Category disclosure (locked requirement): "Categories are derived by TrustLens from listing taglines using fixed keyword rules — OKX.AI does not publish agent categories. The first matching rule wins; agents matching no rule are grouped under 'Other Services'."
7. Version line: "Methodology v{score_version} · data as of {data_as_of}. Any change to the formula, weights, or bands increments the version."
8. Disclaimer paragraph: verbatim `scoring.DISCLAIMER` (same string as footer).

## Accessibility Contract

- `<html lang="en">`; CJK names render via the CJK members of the font stack

- Focus visible on every interactive element (accent outline spec above)
- Badge SVG: `<title>` + `role="img"` + `aria-label`; embed snippet `alt` text: "TrustLens grade {grade}, score {score}"

<decisions>
## Implementation Decisions

### Repo & module layout (locked)
- Package `indexer/` with `python -m indexer.refresh` as the entry point (brief-specified invocation)
- Seed CSV already at `data/okx-marketplace-census-2026-07-10.csv` (committed; do not modify it)
- SQLite DB at `data/trustlens.db` — gitignored, regenerated by refresh
- Stdlib only for this phase: `csv` (handles quoted multiline taglines) + `sqlite3` + `unicodedata`; open CSV with `encoding="utf-8-sig"`. No SQLAlchemy (project research: 272 rows, connection-per-operation)

### Parsing rules (locked — silent-corruption traps from research)
- Sold: `"539 sold"` → 539; `"1.55K sold"` → 1550 (K/M suffixes, case-insensitive, strip commas); missing/blank → 0
- Price: `"0.01 USDT"` → 0.01; subscript-zero notation `"0.0₄15 USDT"` → 0.000015 (U+2080–U+2089 subscript N = N zeros after "0."), `"0.0₅1"` → 0.000001; keep the raw string in a `price_raw` column for display
- Rating: valid ratings are 0–5; if the rating cell holds a price-like value AND `positive` is empty (the shifted-column census artifact), store rating = NULL and positive_pct = NULL — "unrated", never a false 0.01 rating
- Positive: `"92.86% positive"` → 92.86; `"100% positive"` → 100.0; empty → NULL
- Names: preserve unicode exactly; additionally store an NFKC-casefolded lookup key so `这个能吃吗？` and id `"3345"` both resolve later (MCPS-04 depends on this)
- Any unparseable row: log a warning with row number, skip the field (NULL) rather than crash; never silently coerce

### Schema (locked semantics, planner refines DDL)
- `agents`: id TEXT PRIMARY KEY, name TEXT NOT NULL, name_key TEXT (NFKC-casefolded, indexed), category TEXT NOT NULL, tagline TEXT, price_usdt REAL, price_raw TEXT, sold INTEGER NOT NULL DEFAULT 0, rating REAL NULL, positive_pct REAL NULL, first_seen TEXT NOT NULL, last_seen TEXT NOT NULL (ISO-8601 UTC)
- `snapshots`: autoincrement id, agent_id, captured_at, price_usdt, sold, rating, positive_pct, source TEXT ('census' now, 'scrape' in Phase 5) — one row per agent per refresh
- Refresh is idempotent: upsert agents (first_seen preserved, last_seen updated), always append snapshots
- WAL mode on; a fresh DB is created if absent

### Category derivation (locked — resolves the research blocker)
- The census has NO category column. Derive a deterministic category from name+tagline keyword rules, in code, with a fixed bucket set (roughly: Market Data & Analytics; Trading & DeFi; Security & Trust; Sports & Prediction; Lifestyle & Health; Social & News; Developer Tools & Infra; Other Services). First-match-wins over an ordered keyword table = stable and testable
- Category column stores the derived value; a `category_source` column ('derived' now, 'listed' when Phase 5 scraper finds the real one) keeps the methodology honest — the leaderboard/methodology page will disclose "categories derived from listing text"
- Exact keyword table is Claude's discretion but MUST be deterministic, documented in code, and produce no empty categories

### Determinism & testing (locked)
- All timestamps injectable (`now: datetime | None` parameter pattern) — project research: clock injection is what keeps the coverage gate stable
- Fixture-based tests for every edge case above (K-suffix, subscript price, shifted rating, multiline tagline, CJK name); a full-census integration test asserting exactly 272 agents load with zero network
- Indexer tests live in `tests/`; the ≥90% coverage gate applies to `scoring/` (Phase 2), not here — but parser edge cases must have explicit fixtures regardless

### Git & conduct (locked)
- Commits authored by the user's git identity only; NEVER any AI attribution (Co-Authored-By, "Generated with Claude Code", etc.)
- Conventional commits per task (e.g., `feat(01-01): …`)
- Work only inside the trustlens/ repo; the parent folder's CSV/report/prompt are read-only

### Claude's Discretion
- Module file layout inside `indexer/` (e.g., `parse.py` / `db.py` / `refresh.py` split)
- Upsert style (INSERT ... ON CONFLICT vs SELECT-then-write)
- Exact keyword table contents and bucket names (within the locked constraints)
- Logging format (stdlib `logging`)

</decisions>
